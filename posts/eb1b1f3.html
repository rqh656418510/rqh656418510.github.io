IDEA 代码规范检测插件

IDEA 可以使用 CheckStyle-IDEA 插件来检测 Java 代码的规范，它可以保证每位提交者的代码规范都保持一致。值得一提的是，CheckStyle-IDEA 插件只会检测代码的规范，并不能格式化代码。

创建规则文件

特别注意

规则文件 checkstyle.xml 的内容必须与 CheckStyle 的版本互相匹配，否则会影响代码规范检测插件 CheckStyle-IDEA 的运行。

插件安装

  • 1、打开 IDEA 插件市场的界面
  • 2、搜索 CheckStyle-IDEA，点击安装即可

插件配置

导入规则文件

  • 1、打开 CheckStyle 的配置界面（File -> Settings -> Tools -> Checkstyle）
  • 2、选择 Checkstyle 的版本为 8.39，这里的版本号必须与 checkstyle.xml 规则文件的内容相互匹配
  • 3、在界面上点击配置文件的添加按钮，配置描述可随便填写（例如 Custom Checks），然后选中项目里的 checkstyle.xml 规则文件，点击下一步和完成
  • 4、在界面上勾选刚刚添加的配置文件

插件使用

  • 1、在 IDEA 界面内打开任意一个 Java 源文件
  • 2、打开 IDEA 界面底部的 CheckStyle 操作面板，点击左侧的 绿色三角形 按钮，这样就可以检查 Java 代码的规范

值得一提的是，CheckStyle 除了可以检测单个 Java 源文件的代码规范，还支持检测整个 Maven 模块（Check Module）或者整个项目（Check Project）的 Java 代码规范。

Maven 代码格式化插件

Spring Java Format 插件集提供了一款格式化 Java 代码的 Maven 插件，默认使用 Spring 的代码规范，插件的运行依赖于 JDK 11+。在项目里配置好代码格式化的 Maven 插件后，可直接运行命令格式化项目代码： mvn spring-javaformat:apply 或者 ./mvnw spring-javaformat:apply。当然也可以在 IDEA 中安装 Spring-Javaformat 插件进行自动格式化，详细的使用教程请看 官方文档。

基础使用

现在切换到项目所在的目录下，就可以使用 mvn spring-javaformat:apply 或者 ./mvnw spring-javaformat:apply 命令格式化 Java 代码了。
